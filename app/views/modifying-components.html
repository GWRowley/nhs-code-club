There may be times when you are building a prototype and you want to add a feature to a page but there is not a relevant component in the toolkit.

However, there may be other government design systems that do have a relevant component, so in this section of the tutorial we will look at how you can potentially modify another services component to work with the NHS toolkit.

A good place to start to identify a component is this design systems list (opens in a new tab).

Modifying a component

For this part of the tutorial, I have decided to modify the Ministry of Justice timeline component (opens in a new tab).

Stage one

  • take the HTML provided to us by the MOJ and paste it into our desired template
  • replace the moj prefix on the HTML class names to nhsuk or nhsbsa (something relevant). For example change moj-timeline to nhsuk-timeline

Stage two

You will notice how the component looks nothing like it did on the MOJ website, this is because we have no CSS. So next we will need to grab the CSS.

There are two ways of going about this, you can either use developer tools in the browser and manually pick out the styles, or alternatively and the preffered method would be to refer to the GitHub repository for the relevant front end toolkit, which will ensure we don't miss anything we need.

  • locate the relevant GitHub repository, which is MOJ Frontend (opens in a new tab), and browse to the relevant directory (in this case /src/moj/components/timeline)
  • Open the associated SASS file by clicking on the file name (_timeline.scss)
  • Click on the button titled 'Raw', then select all and copy.

Stage three

As briefly mentioned in the introduction section of the tutorial, we will now look at creating our own SASS file.

Whilst not strictly necessary, you can just paste it into app/assets/sass/main.scss creating your own file will make it easier to locate and edit in the event you need to make changes afterwards.

  • Create a new file in the folder app/assets/sass/components and name it _timeline.scss or something relevant
  • Paste all the code we previosuly copied into this file, and save.
  • Like we did in stage one, change the prefixes from moj to nhsuk or whichever prefix you chose in stage one
  • We will then need to carefully go through and replace and SASS variables such as $govuk-brand-colour that don't exist in our toolkit to something that will work with the NHS brand, in this instance I chose $color_nhsuk-blue
  • You may also notice references to the Gov toolkit in this component exmaple, such as govuk-spacing(4) which can be replaced with pixel values and also @include govuk-font($size: 19, $weight: bold); which I will replace with 3 lines of CSS, one for the Frutiger font family, one for font size and one font weight.
  • We then need to import this newly created stylesheet into our main stylesheet located at app/assets/sass/main.scss. To do this on line 7 we would write @import 'components/timeline'; and then save.

What will happen now is the gulp script which is running when we ran npm run watch in the terminal when setting up this project will see the change we made, import all the SASS into one file, then convert the SASS into CSS and output it into a file in the public folder to be used by the layout.html template.

As mentioned in the introduction section of this tutorial, CSS stands for cascading stylesheets - hence why we put our new code on line 7 - below the other two imported style sheets to ensure that none of our code is overwritten, however unlikely thanks to our BEM naming conventions.

Stage four

As you can see, there are just a few bits of tidying up to be done, for example the obvious one is the button in the first timeline item. This is a Gov button, so instead of modifying their component you would just replace with the NHS toolkit button. Their may be other little bits such as gov class names for links etc. you just need to look out for, but hopefully this gives you a good idea of how to modify other services toolkits.
