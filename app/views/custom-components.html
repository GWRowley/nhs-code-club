There may also be a time when you have an idea for a component, and there isn't something already out there for you to use.

If that is the case you may opt to build a custom component of your own. To do so is very similar to modifying an existing component.

For prototyping purposes as its likely unecessary, and for the purpose of this tutorial we will only look at the HTML side of a custom component rather than Nunjucks.

For this part of the tutorial I am going to a build a very simple 'notification card' similar to what you might see when you get a notifcation on Facebook on a desktop.

Stage one

Stage one would be to create a basic 'sketch' of my component:

Stage two

The next stage will be to build the structure in HTML.

As you can see, if you break down the above design, it will need an outer 'container' and two 'inner containers'. One will act as the 'header' which contains notification title, time, and close icon. The other will be the 'body' which will contain information about the notification. Below is an example of how I would build the structure of the component.

	
	<div class="nhsuk-notification-card">
		<div class="nhsuk-notification-card__header">
			<p class="nhsuk-notification-card__title">Notification</p>
			<p class="nhsuk-notification-card__time">10 mins ago</p>
			<button type="button" class="nhsuk-notication-card__close" aria-label="Close">
				<svg class="nhsuk-icon nhsuk-icon__close" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24" aria-hidden="true" focusable="false" width="27" height="27">
					<path d="M13.41 12l5.3-5.29a1 1 0 1 0-1.42-1.42L12 10.59l-5.29-5.3a1 1 0 0 0-1.42 1.42l5.3 5.29-5.3 5.29a1 1 0 0 0 0 1.42 1 1 0 0 0 1.42 0l5.29-5.3 5.29 5.3a1 1 0 0 0 1.42 0 1 1 0 0 0 0-1.42z"></path>
				</svg>
			</button>
		</div>

		<div class="nhsuk-notification-card__body">
			<p>George added a new patient.</p>
		</div>
	</div>

As you may have noticed, I have used the close icon provided by the NHS toolkit, the next step would be to look at other components similar in style I can reference to make sure my component is on brand and consistent with the look of the other components.

Stage three

The next stage will be style the component accordingly.

My initial design is similar to the existing card component, so will use that for inspiration and take some of the styles for that component to use as my own. Like the modifying a component section of this tutorial first we must do the following:

  • create a SASS stylesheet inside of the assets/sass/components directory, I will call this _notification-card.scss
  • import the newly created stylesheet into assets/sass/main.scss

Once we have a dedicated stylesheet we can begin adding our own styles. As mentioned earlier, I can use the GitHub repository for the card component to take the bits I need and build upon it accordingly.

Another important note is making sure it works across all devices with the use of media queries.
