What is CSS?

CSS stands for cascading style sheets, and in the simplest terms if I use the analogy of a website being a house, the brick work and beams are the HTML (the structure of the house / page); CSS is the paint work and furnishings, essentially what 'makes the house a home'. CSS is what makes a website 'look nice'.

The cascading part - again in simple terms means that it reads from top to bottom, cascading down. Meaning if you have two classes of the same name, the property at 'the bottom' will be used. Using the example below if you applied the class nhsuk-example-text to a paragraph tag, the colour of the text would be blue as the bottom version of the class has overwritten the one above.


      .nhsuk-example-text {
        color: green;
      }

      .nhsuk-example-text {
        color: blue;
      }

SASS vs CSS

SASS is essentially a 'more powerful' version of CSS. SASS being a pre-processor of CSS. Syntax wise they are mostly the same, with the exception of 'powerful' features in SASS such as variables. SASS also allows you to 'nest' your code, which in my opinion leads to quicker development, cleaner code, and easier to maintain.

Another major difference between the two is the file extension, SASS uses .scss whereas CSS uses the .css extension.

When you write and save your SASS file, what essentially happens is (depending on what task runner / npm package used) the SASS is then converted to CSS, and also often concatenated into one file, and minified for better performance.

Later on in this 'tutorial' will be details on creating your own SASS file, and more detail on the above.

Using the house analogy again from earlier, here is a very basic example of SASS vs CSS, firstly we will look at how you might style a house using CSS:


      .house {
        background-color: #FFA500;
      }

      .house .door {
        background-color: #000;
      }

      .house .windows {
        background-color: #FFF;
      }

If you were to write the above example in SASS you might write something like this:


      $orange: #FFA500;
      $black: #000;
      $white: #FFF;

      .house {
        background-color: $orange;
        
        .door {
          background-color: $black;
        }

        .windows {
          background-color: $white;
        }
      }

With the use of variables in the second example for the colours, maybe later on you wanted to add a garage, you wouldn't have to remember the hex code for the colours simply the variable name, which in the long term makes developing quicker and easier.

The nesting element of SASS I believe as it's visually similar to how you would write the HTML makes it easy to follow, and target elements for styling easier.

For a much more in depth and better explained version of the above I would recommmend looking at the SASS official guide (opens in a new tab). Some of the other features will also be discussed further on in this tutorial.
